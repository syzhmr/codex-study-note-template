# 手書きノート証明補完方針

このファイルは，ユーザー作成ノートから証明の行間をどのように読み取り，完成版本文へどのように移すかを記録する。
目的は，章を進めるごとに作業者が賢くなるように，ユーザーの証明の読み方，分割の仕方，補足の置き方を再利用可能な型として蓄積することである。

この方針も軽量化のためではなく，再現性のために使う。
本文改訂では `NOTE_POLICY.md` と `HANDWRITTEN_INDEX.md` を優先し，証明の埋め方を決める段階でこのファイルを読む。
手書きノートがない範囲の自律証明と，完成版 PDF と参考文献を確認する独立査読でも，このファイルに蓄積した証明補完の型を判断基準として参照する。

## 役割

- `HANDWRITTEN_INDEX.md`: どのノートページ・画像を見るかを決める台帳。
- `NOTE_POLICY.md`: 完成版ノート全体の編集方針。
- `HANDWRITTEN_PROOF_POLICY.md`: 手書きノートから証明補完の型を抽出し，次の章・小節へ転用するための方針。

## 自律証明と独立査読での使い方

- 自律証明では，対応する手書きノートがない場合でも，このファイルに蓄積した型を使って，定義への戻り方，中間主張の切り出し方，図式・計算・有限段階への帰着の粒度を決める。
- 自律証明では，主テキストが短いことを本文も短くしてよい理由にしない。
- 独立査読では，完成版 PDF の引用，標準事実，証明省略の妥当性を確認するために参考文献群を見てよい。
- 独立査読で「手書きノートなら分けていたはずの中間主張」が本文に埋もれていると判断した場合は，修正文案を `REVISION_CANDIDATES.md` に記録する。

### 独立査読で参照できる資料

| 資料 | 扱い |
|---|---|
| 完成版 PDF | 主対象として読む。論理の流れ，説明の十分性，証明の追跡可能性を確認する。 |
| 参考文献群 | 引用，標準事実，証明省略の妥当性を確認するために参照してよい。 |
| `HANDWRITTEN_PROOF_POLICY.md` | 比較資料ではなく，証明密度と行間の評価基準として参照する。 |
| 主テキスト | 見ない。元原稿からどれだけ離れたかは通常査読で確認する。 |
| 手書きノート | 見ない。密度比較は通常査読で確認する。 |
| 改訂履歴・作業ログ | 見ない。読者として完成版 PDF と参考文献だけから判断する。 |

## 基本原則

1. 手書きノートは補足資料ではなく，ユーザーが証明を追うときに必要だと判断した行間である。
2. 完成版本文は，手書きノートを読まなくても同じ証明を追えるようにする。
3. 証明を整理することを，証明を短くすることと混同しない。
4. 手書きノートにある中間計算，図式，局所的な主張，参考文献候補，赤字・青字の注意は，まず本文へ入れる候補として扱う。
5. 本文へ入れない場合は，どこで吸収したか，後続で扱うか，数学的に不要かを作業記録に残す。
6. 主テキストの番号付き定理・命題・補題番号は原則として増やさない。
7. 必要な中間主張は，番号なしの `claim`，`lem`，`prop`，`recall`，`localdfn` など既存環境で切り出す。
8. 地の文，注意，例の中でも，実質的な数学的主張を使うなら必要に応じて主張として切り出し，直後に証明または確認を書く。
9. 主テキストに元から見出しとして置かれていない補助 `claim` は，原則として長い題名を付けず「主張」だけにする。
10. 地の文から切り出す `claim` は，対象，写像，仮定，結論を主張文の中で明示する。
11. 証明内の `claim` は直前の証明文脈に依存してよいが，証明内の局所的な `lem` は独立した命題として読めるように仮定を明示する。

## 証明補完の手順

各命題・補題・定理・重要な構成について，次の順序で確認する。

1. 主テキストの証明骨格を固定する。
   - 仮定，結論，記号，直前で使える定義，後続で使う結論を短く列挙する。
   - 主テキストまたは手書きノートが証明を複数の役割に分けている場合は，その区切りを作業用の証明段階として記録する。
   - `obvious`, `easy`, `standard`, `it follows` などで圧縮している箇所を拾う。
   - 証明内で使う知識が，既に本文，注意，定義，命題，補題，定理，主張，例で示されていないか確認する。
2. `HANDWRITTEN_INDEX.md` で対応ページまたは画像を確認する。
3. 手書きノートの追加要素を分類する。
   - 証明の必須ステップ
   - 局所的な Claim/Lemma
   - 計算・評価・等式変形
   - 図式・写像の対応
   - 定義への戻り
   - well-defined 性，連続性，同変性，単射性，全射性，同型性，閉性，完備性の確認
   - 赤字の修正・注意
   - 青字の補足・疑問
   - 参考文献名，URL，タグ，定理名，補題名
   - 色付き矢印，引き出し線，丸囲み，下線，括弧などで示された参考文献候補や根拠箇所
4. 本文に入れる形を決める。
   - 大きな証明段階としてまとめるか。
   - その証明段階の内部で使う一文の補足か。
   - 既出結果への参照か。
   - 新たに示す `claim` や局所的な `lem` として切り出すか。
   - 地の文由来の `claim` なら仮定を環境内に明示できているか。
   - 証明内の `claim` なら，直前の証明文脈で仮定が十分に固定されているか。
   - 証明内の局所的な `lem` なら，独立した命題として読めるだけの仮定が環境内にあるか。
5. 本文案を書いた後，手書きノートと照合する。
   - 手書きノートにある中間ステップが本文で消えていないか。
   - 消えた場合，本文のどこで吸収されたか。
   - 吸収できていない場合，本文へ戻すか，未解決点にする。
6. 完成版 PDF を主対象として読んで確認する。
   - 作業資料名が本文に出ていないか。
   - 読者が定義に戻って同じ証明を再現できるか。

## 手書き要素ごとの扱い

| 手書きノートの要素 | 本文での扱い |
|---|---|
| 赤字の修正 | 誤読防止または修正として必ず確認する。本文で赤字とは書かない。 |
| 青字の補足 | 行間補足として原則反映する。疑問符付きなら根拠確認後に反映する。 |
| 図式 | 図式だけを置かず，射の定義，向き，可換性を文章で説明する。 |
| 証明の必須ステップ | 主テキスト・手書きノートから読み取れる証明段階として扱う。完成版の見出しとして機械的に `Step` を置かない。 |
| 長い計算 | 途中式を落とさない。役割が変わる箇所で自然な段落，接続文，必要な局所主張に分ける。 |
| 既出の注意・補題・主張 | 同じ内容を再証明せず，参照マクロで既出箇所へ飛ばす。 |
| 小さな中間主張 | 後で使う，または確認が必要なら `claim` で切り出す。 |
| 局所的な補題 | 証明の構造を支えるなら `lem` または番号なし補題として置く。 |
| 定義の再掲 | 初出から離れているなら `recall` または短い本文で戻る。 |
| その場だけの用語定義 | `localdfn` を検討する。 |
| 参考 URL・文献名 | `references/notes/README.md` の探す順序で照合し，未判読なら未解決点にする。 |
| 色付き矢印の参考候補 | 始点・終点，色，周辺語句を記録して照合する。 |
| 矢印先の数学的主張 | 出典候補ではなく数学的内容として抽出し，必要なら `claim`，`lem`，`rmk`，`recall` などで本文へ反映する。 |
| 斜線や取り消し | 取り消された流れを採用しない。採用された追記側を優先する。 |

## Claim/Lemma に切り出す基準

次のいずれかに当たる場合は，地の文だけで流さず，局所的な主張として切り出すことを検討する。

1. 証明の後半で再利用する。
2. その主張がないと読者が次の行を追えない。
3. well-defined 性，連続性，同変性，閉性，完備性，単射性，全射性，同型性を確認している。
4. 有限段階から極限へ戻す。
5. 商，核，像，固定部分，剰余環，係数拡大，テンソル積，双対を扱う。
6. 手書きノートが明示的に Claim/Lemma 風に分けている。
7. 地の文や注意の中で「これは同型である」「これは閉である」「これは選択によらない」などを使っている。

切り出した主張は，必要なら直後に証明を書く。
「後で示す」と置いた場合は，後続で必ず回収する。

## 証明復元表の形式

証明を本文へ移す前に，命題・補題・定理ごとに次の表を作る。
表は作業用であり，本文へそのまま載せる必要はない。

| 項目 | 記録する内容 |
|---|---|
| 対象 | 章，小節，命題・補題・定理番号，対象の短い説明。 |
| 主テキストの骨格 | 主テキストに明示されている仮定，結論，証明の流れ。 |
| 手書きノートの追加ステップ | 手書きノートで補われている定義への戻り，中間計算，図式，Claim/Lemma。 |
| 地の文・注意由来の主張 | 切り出す理由，直前の接続文，反映先，証明または確認の有無，主張文内に明示する仮定。 |
| 証明内の局所主張・補題 | 属する証明段階，直前の宣言文，文脈に預ける仮定，環境内に明示する仮定，局所証明の終了位置。 |
| 参考文献で確認した根拠 | 文献名，節，定理・命題・補題番号，ページまたはタグ，用途。 |
| 色付き矢印で示された参考候補 | 始点・終点，色，周辺語句，判読状況，照合結果。 |
| 本文へ入れる補足 | 完成版本文へ入れる定義，文脈，局所的主張，証明ステップ。 |
| 本文へ入れない箇所 | どこで吸収したか，標準事実として引用するか，未解決点にするか。 |
| 反映位置 | TeX 上の節，命題，証明，注意，脚注などの反映先。 |
| 検査結果 | 完成版 PDF を主対象として追えるか，参考文献確認が必要な省略が残るか。 |
